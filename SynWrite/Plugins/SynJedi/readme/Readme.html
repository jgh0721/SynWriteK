Intro

Plugin for SynWrite editor. Intelligent auto-completion and find-id for Python lexer.
Based on awesome Jedi library.


How to install

Latest SynWrite supports plugins auto-installing. Just open "SynJedi.rar" file in SynWrite, you'll see plugin installation prompt. Confirm it, then wait until "unrar" window finishes its work.

Note: if you have renamed "Python" lexer, correct also lexer name in "SynPlugins.ini" file, using command "Options -- Advanced -- Edit file SynPlugins.ini".


How to use

Notes

  • On first call, plugin runs server window. This window is minimized, you can close it any time after work in editor is ended. Server runs slowly (2-4 sec) but works fast.
  • If plugin fails, you should see message in SynWrite statusbar, e.g. "[SynJedi] No results".

1) Auto-completion

Open any Python script, place caret after "objname." (where objname is some object instance), and press Ctrl+Space (default auto-complete shortcut). You should see message in SynWrite statusbar "Trying auto-complete: .........", then you should see, after a short delay, auto-completion list for your objname.

2) Function hints

Place caret after "objname.funcname(" and press Ctrl+Shift+Space (default func-hint shortcut), after few seconds delay you should see function hint, above caret.

3) Find symbol definition

Right-click any identifier (module name, class name, function name, variable), to call editor context menu, and choose menu item "Find ID" (if menu item is disabled, you didn't configure "SynPlugins.ini" section [FindID]). If id declaration can be found by Jedi library, you'll see "Found" message in statusbar, and caret placed on id declaration in some file.

Also you can use Ctrl+Alt+click, instead of context menu.


How to specify search paths

You can specify additional folders, for your custom Python libraries, using one or both of methods:

  • list folders in SynWrite options dialog, "Path Environment" tab;
  • list folders in current SynWrite project's properties dialog (if you have different folders for each project).

Notes:

  • If you have changed search folders, server window must be closed, to apply this setting to server.
  • SynWrite feature "append '*' to search folder to search in all subfolders" isn't supported here.


Config file

You may create file "SynJedi.ini" in the SynJedi folder and write there few options in section [ini]:

  • datadir: Specifies folder where plugin finds its folder "server".
  • hints (bool, 0 or 1): Enables to show function hints.
  • server_port: Specifies TCP port for server process. Default is 11112.
  • server_hide (bool): Enables to hide server window (otherwise window is minimized).
